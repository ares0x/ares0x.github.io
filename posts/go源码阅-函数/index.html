在 Go 中，函数是一等公民。

调用惯例

函数的通用语法：function(arg1, arg2)

Go 使用栈传递参数和接收返回值，所以只需要在栈上多分配一些内存即可返回多个值

参数传递

  • 传值：函数调用时会对参数进行拷贝，被调用方和调用方两者持有不相关的两份数据
  • 传引用：函数调用时传递参数的指针，被调用方和调用方持有相同的数据，任意一方做吃的修改都会影响另一方。

Go 语言是传值的方式，无论是传递基本类型，结构体还是指针，都会对传递的参数进行拷贝。

func myFunction(i int, arr[2]int) {
    i = 10
    arr[1] = 34
}

func main() {
    i = 20
    arr := [2]int{66,77}
    myFunction(i,arr)
}
// i 和 arr 在 main 函数中的地址和在 myFunction 中的地址完全不同，即在参数传递时进行了拷贝
// 在 myFunction 中的修改仅仅影响了当前函数，不会对 main 函数中的值产生影响

所以，在 Go 中整型和数组类型是值传递的，在调用函数时会对内容进行拷贝。

func myFunctionStructAndPointer(a MyStruct, b *MyStruct)  {
	fmt.Printf("bofore modify  - a=(%d, %p) b=(%v, %p)\n", a, &a, b, &b)
	a.i = 31
	b.i = 41
	fmt.Printf("in my_function - a=(%d, %p) b=(%v, %p)\n", a, &a, b, &b)
}

func main() {
	a := MyStruct{i: 30}
	b := &MyStruct{i: 40}
	fmt.Printf("before calling - a=(%d, %p) b=(%v, %p)\n", a, &a, b, &b)
	myFunctionStructAndPointer(a, b)
	fmt.Printf("after calling  - a=(%d, %p) b=(%v, %p)\n", a, &a, b, &b)
}
// before calling - a=({30}, 0x140000200d8) b=(&{40}, 0x1400000e028)
// bofore modify  - a=({30}, 0x140000200f0) b=(&{40}, 0x1400000e038)
// in my_function - a=({31}, 0x140000200f0) b=(&{41}, 0x1400000e038)
// after calling  - a=({30}, 0x140000200d8) b=(&{41}, 0x1400000e028)

结构体和结构体指针在调用函数后的指针地址已经修改（做了拷贝），修改结构体指针中的值会影响原始值。

  • 传递结构体时，会拷贝结构题中的全部内容
  • 传递结构体指针时，会拷贝结构体指针

结构体在内存中也是一块连续的内存空间，指向结构题的值很也是指向结构体的首地址。 将指针作为参数传入函数时，函数内部会复制指针，即会同时出现两个指向原有内存空间的指针，所以 Go 中的指针也是传值。

可以推测，Go 语言在传递参数时使用传值的方式，接收方收到参数时会对这些参数进行复制。所以，在传递数组或者占用非常大的结构体时，应该尽量使用指针作为参数类型来避免发生数据拷贝进而影响性能。

规则：

  • 通过堆栈传递参数，入栈顺序是从右往左，参数的计算是从左往右（先进后出）
  • 函数返回值通过堆栈传递并由调用者预先分配内存空间
  • 传值
