编译过程

编译器的前端和后端，分别负责词法分析，语法分析，类型检查，中间代码生成和目标代码生成，优化等功能。其中编译后端就是将前端生成的中间代码翻译为目标机器可以运行的二进制机器码。

Go 的编译器在逻辑上被分为四个阶段：词法与语法分析，类型检查，通用 SSA （静态单赋值）生成和机器代码生成。

词法分析和语法分析

词法分析的作用是：使用词法解析器（lexer）解析源代码文件。它将文件中的字符串序列转换为 Token 序列，方便后续的处理和解析。

语法分析的作用是：将词法分析器生成的 Token 序列按照顺序解析，并将其按照编程语言定义好的文法（Grammer）规约，每个 Go 的源代码会被归纳成一个 SourceFile 的结构。 词法分析器生成的 Token 序列不包含空格，换行符等。语法分析器会把 Token 序列转换成语法树。

类型检查

类型检查阶段会对语法树中定义和使用的类型进行检查。通过对整棵抽象语法树（AST）的遍历，保证节点不存在类型错误。所有的类型错误和不匹配会在该阶段被暴露出来。在该阶段还会展开和改写一些内建函数。（make -> runtime.makeslice OR runtime.makechan）

强类型的编程语言在编译期间会有严格的类型限制，编译器会在编译期间发现变量赋值，返回值和函数调用时的类型错误。（Go 强类型）

静态类型检查

静态类型检查是基于对源代码的分析来确定运行程序类型安全的过程。静态类型检查可以帮助我们在编译器发现程序中的类型错误。

数组的长度在类型检查期间确定。map 与 slice 类似。 在类型检查阶段之前，无论是创建切片，哈希还是 Channel 用的都是 make 关键字。在类型检查阶段会根据创建的类型，将 make 替换为特定类型的函数。

中间代码生成

在经过类型检查后，可以认为当前文件中的代码不存在语法错误和类型错误的问题，Go 编译器会将输入的抽象语法树转换为中间代码。其中 compileFuncions 会编译整个 Go 语言中的全部函数，并发执行的 Goroutine 会将所有函数对应的抽象语法树转换为中间代码。

在生成中间代码之前，编译器会替换抽象语法树中节点的一些元素，如 panic 和 recover 两个内奸函数转换为 runtime.gopanic 和 runtime.gorevover 两个真正的运行时函数，new 会被转换为 runtime.newobject
