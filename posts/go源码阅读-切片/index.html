切片：动态数组，长度不固定，可以向切片中追加元素，会在容量不足时自动扩容

  • 切片在编译期间生成的类型只包含切片的元素类型(编译期确定类型，运行时动态获取)

切片引入了一个抽象层，提供了对数组中部分连续片段的引用。作为数组的饮用，可以在运行期间修改它的长度的范围。当底层数组长度不足时触发扩容。

编译器在编译期间简化了获取数组大小，读取数组中的元素等操作：数组的内存固定且连续，多数操作会直接读写内存的特定位置。切片在编译期间生成的类型只包含切片的元素类型，在运行时才会确定内容。所有的操作依赖 Go 的运行时。

初始化

三种初始化方式：

1.使用下标

通过下标的方式获得数组或者切片的一部分

1
2

	var arr = [...]int{1,2,3}
	var slice = arr[0:1]

这种方式，编译器会将器转换为 OpSliceMake 操作，该操作会接受四个参数创建新的切片：元素类型，数组指针，切片大小和容量。

使用下标初始化切片不会拷贝原数组或切片中的数组，只会创建一个指向原数组的切片结构，所以修改新切片的数据也会修改原切片。

1
2
3
4
5
6

func main() {
	var arr = [...]int{1,2,3}
	var slice = arr[0:1]
	slice[0] = 9
	fmt.Print(slice,arr) // [9] [9 2 3]
}

2. 使用字面量初始化切片

1

slice1 := []int{2,3,4}

过程：

1
2
3
4
5
6

var vstat [3]int
vstat[0] = 1
vstat[1] = 2
vstat[2] = 3
var vauto *[3]int = new([3]int)
*vauto = vstat
slice := vauto[:]
  1. 根据切片中的元素数量对底层数组的大小进行推断并创建一个数组；
  2. 将这些字面量元素存储到初始化的数组中；
  3. 创建一个同样指向 [3]int 类型的数组指针；
  4. 将静态存储区的数组 vstat 赋值给 vauto 指针所在的地址；
  5. 通过[:]操作回去底层使用的 vauto 切片

[:] 操作是创建切片最底层的一种方法。

3. 使用关键字 make 创建切片

1

slice2 := make([]int, 2)

使用关键字 make 创建切片，需要运行时的支持。 make 方法底层调用 OMAKE，然后转化为 OMAKESLICE

中间代码生成时会依据下面两个条件转换为 OMAKESLICE 类型的节点：

  1. 切片的大小和容量是否足够小；
  2. 切片是否发生了逃逸，最终在堆上初始化。

// TODO 内存逃逸

当切片非常大或则发生逃逸时，运行时需要在堆上初始化切片。 如果没有逃逸或者数量较小，make([]int,3,4) 会被直接转换成

1
2

var arr [4]int
n := arr[:3]

makeSlice 方法主要用来计算切片占用的内存空间，并在堆上申请一片连续的的内存。如果在切片创建过程中发生一下错误，会触发运行时错误并崩溃

  1. 内存空间的大小发生溢出
  2. 申请的内存大于最大可分配内存
  3. 传入的长度小于0或长度大于容量

创建最后会调用 mallocgc 用于申请内存。如果对象的 size 较小会直接在 Go 调度器中的 P 结构中初始化，大于 32KB 会在堆上初始化。 构建运行时结构体的 reflect.SliceHeader 的工作交给了 runtime.makeslice 的调用方。该函数仅会返回指向底层数组的指针，调用方会在编译期间构建切片结构体。

访问元素

len 和 cap 会在 SSA 生成阶段被转化为 OpSliceLen 和 OpSliceCap 访问切片中元素使用的 OINDEX 操作也会在中间代码生成期间转换为堆地址的直接访问。

  • 切片的操作基本都是在编译期间完成的。

追加和扩容

如果使用 slice = append(slice,1,2,3) 语句，append 后的切片会覆盖原切片。 如果覆盖原有的切片，则不需要担心发生拷贝影响性能。

当切片容量不足时，会调用 runtime.growslice 函数为切片扩容（扩容是为切片分配新的内存空间并拷贝原切片中的元素的过程）

切片的扩容策略：

  1. 如果期望容量大于当前容量的两倍，就会直接使用期望容量； 否则（期望容量不大于当前容量的两倍）
  2. 如果当前切片的长度小于 1024 就会直接将容量翻倍；
  3. 如果当前切片的长度大于 1024 就会每次增加 25% 的容量，直到新的容量大于期望容量。

上述步骤会确定大致容量，还要根据切片中元素大小对齐内存。如果数组中元素所占字节大小为 1，8，2 的倍数时，需要向上取整。取整时会使用 class_to_size 数组，使用该数组中的整数可以提高内存分配的笑了并减少碎片。

growslice 函数最终会返回一个新的切片，其中包含了新的数组指针，大小和容量，这个返回的三元组最终会覆盖原切片。

切片拷贝

copyany 两种： 编译期间拷贝 运行时拷贝

无论是编译期间拷贝还是运行时拷贝，两种方式都会通过 runtime.memmove 将整块内存的内容拷贝到目标的内存区域。
