# Go源码阅读 字符串


字符串往往被看成一个整体，但它实际上时一片连续的内存空间，可以将其理解为一个**由字符组成的数组**。Go 中字符串是一个**只读**的字节数组。
Go 语言只是不支持**直接修改** string 类型变量的内存空间，可以通过 string 和 []byte 类型之间反转实现修改
- 1. 将这段内存拷贝到堆或栈上；
- 2. 将变量的类型转换成 []byte 后修改字节数据；
- 3. 将修改后的字节数组转换为 string

## 数据结构
字符串在运行时会使用 StringHeader 来表示，其中包含指向字节数组的指针和数组的大小，跟切片类似，区别是相较于 Slice，少了 Cap。（string 只读切片）
```go
type StringHeader struct {
    Data uintptr
    Len int
}

type SliceHeader struct {
    Data uintptr
    Len int
    Cap int
}
```
字符串是只读的类型，所以不会直接向字符串追加元素改变本身的内存空间，所有在字符串上的写入操作都是通过拷贝实现的。

Go 语言中声明字符串是通过双引号和反引号实现的。
```go
str1 = "ares0x" // 只能用于单行字符串的初始化
str2 = `ares0x` // 反引号可以摆脱单行的限制，而且可以在字符串内部使用双引号
```

## 拼接
可以使用 + 号拼接字符串。addstr 会帮助程序在编译期间选择合适的函数对字符串进行拼接。
 - 1. 如果小于或等于 5 个，会调用 concatstring 等函数
 - 2. 如果超过 5 个，会选择 concatstrings 传入一个数组切片
